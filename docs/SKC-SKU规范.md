# SKC / SKU 编码规范

> 版本：v1.0
> 更新时间：2026-04-25
> 用途：定义 sERP 系统中正式产品的 SKC 和 SKU 编码规则，agent 在分配编码时必须严格遵循本文档。

---

## 一、基本定义

| 术语 | 全称 | 定义 | 示例 |
|------|------|------|------|
| **SKC** | Stock Keeping Class | 产品类（产品主体），一个 SKC 代表一个产品款式 | `WALLET-0001` |
| **SKU** | Stock Keeping Unit | 产品具体规格变体，一个 SKC 下可包含多个 SKU | `WALLET-0001-BEIGE` |

**关系：** SKC 是产品主体，SKU 是具体规格变体。SKU = SKC + 规格后缀。

---

## 二、编码规则

### 2.1 SKC 编码格式

```
品类代码-序号
  ↑4位    ↑4位数字
```

**示例：**
- `WALLET-0001` — 钱包品类第1个产品
- `PHCA-0001` — 手机壳品类第1个产品
- `BACK-0001` — 背包品类第1个产品

### 2.2 SKU 编码格式

```
品类代码-序号-规格
  ↑4位    ↑4位数字 ↑变体特征
```

**示例：**
- `WALLET-0001-BEIGE` — 钱包第1个产品，米色款
- `WALLET-0001-BROWN` — 钱包第1个产品，深棕色款
- `PHCA-0001-BLACK` — 手机壳第1个产品，黑色款
- `PHCA-0001-RED` — 手机壳第1个产品，红色款

### 2.3 规格后缀规则

| 规格类型 | 格式 | 示例 |
|---------|------|------|
| 颜色 | 英文颜色名全大写 | `BEIGE`, `BLACK`, `BROWN`, `RED`, `WHITE` |
| 尺寸 | 单个大写字母 | `S`, `M`, `L`, `XL` |
| 颜色+尺寸 | 颜色-尺寸 | `BLACK-S`, `RED-L` |
| 其他特征 | 英文特征名全大写 | `LEATHER`, `METAL`, `WOOD` |

## 四、去重机制（哈希表）

### 4.1 数据结构

系统维护一个 `data/products.json` 文件，包含已注册编号的哈希表：

```json
{
  "已注册编号": {
    "WALLET-0001": "Travelambo RFID 钱包女式大号双折多卡钱包保护套带拉链口袋信用卡包",
    "WALLET-0002": "另一个钱包产品名称",
    "PHCA-0001": "iPhone 14透明手机壳"
  },
  "产品列表": [
    {
      "skc": "WALLET-0001",
      "skus": ["WALLET-0001-BEIGE", "WALLET-0001-BROWN"],
      "title": "Travelambo RFID 钱包",
      "source_task_id": "collect_d4524658",
      "created_at": "2026-04-25T23:29:30",
      "product_data": { ... },
      "images_dir": "data/collect_collect_d4524658/images"
    }
  ]
}
```

### 4.2 分配流程

```
1. 用户点击"保存为正式产品"
2. 系统读取 docs/SKC-SKU规范.md 获取编码规则
3. 系统读取 data/products.json 中的"已注册编号"哈希表
4. 根据采集数据的品类，从品类映射表获取4位代码
5. 查询该品类已使用的最大序号，+1 生成新序号
6. 检查 SKC 是否在哈希表中唯一
7. 根据采集数据中的变体信息生成 SKU 列表
8. 将新 SKC/SKU 写入哈希表和产品列表
9. 返回 SKC 编号给前端显示
```

### 4.3 唯一性保证

- 每次分配前必须查询 `已注册编号` 哈希表
- 如果生成的 SKC 已存在，序号继续递增直到唯一
- 不允许手动修改已分配的 SKC/SKU

## 五、完整示例

### 示例1：钱包产品（2个颜色变体）

```
采集数据：
  标题: Travelambo RFID 钱包女式大号双折多卡钱包保护套带拉链口袋信用卡包
  平台: amazon
  价格: HKD78.27
  变体: 米色、深棕色

分配结果：
  SKC:  WALLET-0001
  SKU:  WALLET-0001-BEIGE    ← 米色款
        WALLET-0001-BROWN    ← 深棕色款
```

### 示例2：手机壳产品（3个颜色变体）

```
采集数据：
  标题: iPhone 14 透明防摔手机壳
  平台: amazon
  变体: 黑色、白色、透明

分配结果：
  SKC:  PHCA-0001
  SKU:  PHCA-0001-BLACK    ← 黑色款
        PHCA-0001-WHITE    ← 白色款
        PHCA-0001-CLEAR    ← 透明款
```

## 六、注意事项

1. **品类代码必须唯一无歧义**，新增品类时需人工审核
2. **序号同品类递增**，不同品类序号独立（WALLET-0001 和 PHCA-0001 不冲突）
3. **SKU 规格后缀**使用英文全大写，颜色名使用标准英文
4. **已分配的 SKC/SKU 不可修改**，如需废弃需标记状态而非删除
5. **agent 每次分配编码前必须读取本文档**，确保遵循最新规范
