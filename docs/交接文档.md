## 三、模块架构

### 模块关系图

```
main.py (入口)
   │
   ├── app.py (Flask 后端) ← 所有 HTTP API 和路由
   │      │
   │      ├── 图片批量处理 (task_* 数据目录)
   │      ├── 产品采集 (/api/collect/*)
   │      ├── 产品管理 (/api/products/*)
   │      ├── Ozon 品类 (/api/ozon/*)
   │      │     ├── 品类树 (category-tree + 缓存)
   │      │     ├── 品类翻译 (translate-categories)
   │      │     ├── 品类匹配 (match-category → DeepSeek)
   │      │     └── 品类属性 (category-attributes)
   │      ├── Ozon 上架草稿 (/api/listings/*)
   │      ├── 店小秘自动填充 (/api/auto-fill/*)
   │      └── 店铺管理 (/api/stores/*)
   │
   └── collector.py (采集引擎)
          ├── 抓取层: Playwright + requests 双引擎
          ├── 决策层: DeepSeek 图片分类
          └── 执行层: aiohttp 异步下载 + Pillow 转 JPG
```

### 关键数据流

```
采集流程:
  URL → 抓取层 → product_data.json + image_urls
       → DeepSeek 分类 → images_mapping.json
       → 异步下载 → images/*.jpg

品类树翻译流程:
  Ozon API → category_tree.json (缓存)
           → 展平 7998 节点
           → 逐批发 DeepSeek → translations.json (缓存)
           → 前端合并渲染

Ozon 上架流程:
  产品数据 + 品类树 → DeepSeek 匹配品类
                    → 获取品类属性
                    → DeepSeek 填充属性值
                    → 保存上架草稿
```

## 五、当前已知问题

### 🔴 问题 1（优先级最高）："更新品类树"按钮翻译失败

**文件**: `app.py` → `_batch_translate_categories()` + `_run_refresh_in_background()`

**症状**:
- 日志输出 `[品类刷新 第5/25批(Товары для взрослых)] ✅ 成功解析 0 个翻译结果`
- 翻译缓存 `translations.json` 始终为空或只保存了少量数据
- 说明 DeepSeek API 调用成功了（返回了内容），但 JSON 解析失败——未能从中提取出 `{"id": xxx, "name_cn": "..."}` 格式的翻译条目

**当前状态**: 已改为异步后台按一级大类（25批）逐批翻译，每批约 300-500 个品类。实际运行时前 4 批正常，第 5 批开始解析失败。

**可能原因**:
1. **DeepSeek 返回格式不稳定** — 当品类数量较大时，DeepSeek 返回的 JSON 可能不再是最外层的 `{"translations": [...]}` 结构，或返回了 markdown 包裹的代码块，导致 `re.search(r'\{[^{}]*\}', llm_text)` 匹配到不完整的 JSON 块
2. **解析正则过于简单** — `re.search(r'\{[^{}]*\}')` 只匹配最外层第一对花括号，对于深层嵌套的 JSON 会截断内容
3. **ID 类型不一致** — DeepSeek 返回的 `id` 可能与传入的 `id` 类型（str vs int）不匹配

**推荐修复方向**:

方案 A — 改进 JSON 解析（推荐）：
```python
# 替换简单正则，支持多层嵌套和 markdown 代码块
def extract_json(text: str):
    text = text.strip()
    # 去除 markdown 代码块包裹
    if text.startswith("```"):
        text = re.sub(r'^```(?:json)?\s*\n?', '', text)
        text = re.sub(r'\n?```$', '', text)
    text = text.strip()
    # 找到第一个 { 和最后一个 }
    start = text.find('{')
    end = text.rfind('}')
    if start != -1 and end != -1:
        return json.loads(text[start:end+1])
    return None
```

方案 B — 减小每批数量（临时缓解）：
```python
BATCH_SIZE = 200  # 从当前 ~300-500 降低到 200
# 减少品类数以降低输出 token 量
```

方案 C — 增加 max_tokens + 优化 prompt：
```python
"max_tokens": 65536  # 从 32768 翻倍
# 或要求 DeepSeek 逐行输出，降低单次输出量
```

### 🟡 问题 2：品类匹配 DeepSeek max_tokens 偏小

**文件**: `app.py` → `ozon_match_category`，`max_tokens: 512`

当前品类匹配仅返回品类 ID，512 tokens 足够。如需返回更多属性建议，可调大到 1024。

## 七、品类树缓存工作原理

```
Ozon API → 品类树 (7998 节点)
         → category_tree.json (原始缓存)
         → 展平所有节点
         → 检查 translations.json (id → 中文)
         → 未命中 → 分批发送 DeepSeek 翻译
         → 写入 translations.json
         → 前端合并渲染

关键文件:
  data/ozon_cache/{store_id}_category_tree.json
  data/ozon_cache/{store_id}_translations.json
```
