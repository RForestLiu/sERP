# sERP 需求文档

> AI 图像批量处理工具 + 电商产品数据采集工具 + 正式产品管理
> 版本：v1.2
> 更新时间：2026-04-25

---

## 一、产品概述

sERP 是一款面向电商运营的 AI 图像批量处理工具，支持批量导入图片和 Prompt，调用 AI 生图 API 批量生成产品图，以及电商产品数据采集与正式产品管理。

### 目标用户
- 跨境电商卖家
- 需要批量处理产品图的运营人员
- 需要采集和管理电商产品的运营人员

### 核心价值
- 批量上传 + 批量 Prompt 匹配
- 任务管理 + 数据持久化
- 电商产品数据采集（标题、价格、描述、图片）
- 正式产品管理（SKC/SKU 编码自动分配）
- 提升图片处理效率

## 三、功能模块

### 3.1 图片批量处理模块

#### 3.1.1 任务管理（左侧边栏）

| 功能 | 描述 |
|------|------|
| 任务列表 | 显示所有任务，按创建时间排序 |
| 新建任务 | 点击"+新建"按钮创建新任务 |
| 任务命名 | 双击任务名称可编辑重命名 |
| 切换任务 | 点击任务切换到对应任务界面 |
| 删除任务 | 点击"x"删除任务（需确认） |

**数据持久化要求：**
- 每个任务独立存储为一个 JSON 文件
- 文件路径：`tasks/{task_id}.json`
- 任务数据结构：
```json
{
  "id": "abc123",
  "name": "任务名称",
  "created_at": "2026-04-24T10:00:00",
  "json_input": "原始JSON文本",
  "cards": [
    {
      "imageName": "product1.jpg",
      "prompt": "提示词内容",
      "uploadUrl": "/uploads/xxx.jpg",
      "uploadPath": "/uploads/xxx.jpg",
      "resultUrl": "/outputs/abc123/card_0_xxx.png"
    }
  ]
}
```

#### 3.1.2 工具栏（顶部）

| 组件 | 功能 |
|------|------|
| JSON文本框1 | 填写JSON格式的图片配置，高度约96px |
| 拖拽上传区 | 支持拖拽或点击上传多张图片 |
| 拆分提示词按钮 | 解析JSON，将prompt填充到对应的文本框2 |

**JSON格式示例：**
```json
[
  {
    "image_name": "product1.jpg",
    "prompt": "产品图描述..."
  },
  {
    "image_name": "product2.jpg", 
    "prompt": "产品图描述..."
  }
]
```

#### 3.1.3 卡片区域（主内容区）

每个卡片包含：

| 组件 | 功能 |
|------|------|
| 图片框1 | 显示上传的原始图片，支持拖拽/点击上传 |
| 执行生图按钮 | 调用AI API生成图片 |
| 文本框2 | 显示/编辑prompt，约3-4行高度 |
| 图片框2 | 显示生成的结果图片 |
| 下载按钮 | 下载结果图片 |
| 重试按钮 | 重新生成图片 |

**卡片操作：**
1. 单独上传图片到某个图片框1
2. 编辑文本框2的prompt内容
3. 点击"执行生图"调用API
4. 查看结果图片，可下载或重试

#### 3.1.4 图片预览增强

| 功能 | 描述 |
|------|------|
| 悬停大图 | 鼠标移动到图片上显示320x320大图预览 |
| 右键新标签页 | 支持Chrome右键"在新标签页打开图片" |
| 点击跳转 | 点击图片在新标签页打开原图 |

#### 3.1.5 文件管理

| 目录 | 用途 |
|------|------|
| `uploads/` | 上传的原始图片 |
| `outputs/{task_id}/` | 每个任务的生成结果 |
| `outputs/{task_id}/draft/` | 重试后被丢弃的图片 |

**重试逻辑：**
1. 点击"重试"按钮
2. 旧结果图片移动到 `draft/` 子文件夹
3. 生成新图片
4. 更新结果图片URL

### 3.2 采集产品模块

#### 3.2.1 产品数据采集

| 功能 | 描述 |
|------|------|
| 输入网址 | 输入电商产品详情页 URL（支持 Amazon 等平台） |
| 开始采集 | 启动异步采集任务，后台抓取产品数据 |
| 实时进度 | 显示采集进度（抓取中 → 分类中 → 下载中 → 完成） |
| 查看数据 | 弹窗查看产品数据和图片映射表 |
| 打开文件夹 | 打开采集数据所在文件夹 |

**采集数据包含：**
- 产品标题、价格、平台
- 产品描述（About This Item / Product Description）
- 产品属性
- 图片 URL 列表
- 图片下载（按 SKU/场景分类）

#### 3.2.2 正式产品管理

| 功能 | 描述 |
|------|------|
| 保存为正式产品 | 将采集数据保存为正式产品，自动分配 SKC/SKU |
| 状态显示 | 已保存的卡片显示 ✅ 已保存 (WALLET-0001) |
| 去重机制 | 哈希表确保 SKC 编号唯一，重复保存返回 409 |

**SKC/SKU 编码规则（详见 `docs/SKC-SKU规范.md`）：**
- SKC 格式：`品类代码-序号`（如 `WALLET-0001`）
- SKU 格式：`品类代码-序号-规格`（如 `WALLET-0001-BEIGE`）
- 品类代码：4位大写字母，无歧义
- 序号：4位数字，同品类递增

**数据存储：**
- 文件路径：`data/products.json`
- 结构：
```json
{
  "已注册编号": {
    "WALLET-0001": "产品标题"
  },
  "产品列表": [
    {
      "skc": "WALLET-0001",
      "skus": ["WALLET-0001-BEIGE"],
      "title": "产品标题",
      "category": "钱包",
      "source_task_id": "collect_xxx",
      "created_at": "2026-04-25T23:29:30",
      "product_data": { ... },
      "images_dir": "data/collect_xxx/images"
    }
  ]
}
```

## 四、实时持久化

**触发保存的操作：**
- ✅ 创建任务
- ✅ 重命名任务
- ✅ 上传图片
- ✅ 编辑prompt文本框
- ✅ 生成图片成功
- ✅ 切换任务前（自动保存当前任务）
- ✅ 采集任务完成（持久化到 collect_tasks.json）
- ✅ 保存为正式产品（持久化到 products.json）

**防抖机制：**
- 文本编辑使用800ms防抖，避免频繁写入

## 五、API 接口需求

### 5.1 图片批量处理 API

| 方法 | 路径 | 功能 | 请求体 | 响应 |
|------|------|------|--------|------|
| GET | `/api/tasks` | 获取任务列表 | - | `[{id, name, created_at}]` |
| POST | `/api/tasks` | 创建任务 | `{name?}` | `{id, name, created_at, cards}` |
| GET | `/api/tasks/<id>` | 获取任务详情 | - | 完整任务对象 |
| PUT | `/api/tasks/<id>` | 更新任务 | `{name?, data?}` | `{status: ok}` |
| POST | `/api/tasks/<id>/upload_source_images` | 上传图片 | FormData | `{saved: [...]}` |
| POST | `/api/generate` | 生成图片 | `{task_id, card_id, prompt, source_image_path}` | `{url, base64}` |
| POST | `/api/tasks/<id>/save_images` | 保存生成图片 | - | `{moved: [...]}` |
| POST | `/api/tasks/<id>/compress_images` | 压缩图片 | - | `{compressed_count, saved_bytes}` |
| POST | `/api/tasks/<id>/open_folder` | 打开文件夹 | - | `{status, folder}` |

### 5.2 采集产品 API

| 方法 | 路径 | 功能 | 请求体 | 响应 |
|------|------|------|--------|------|
| POST | `/api/collect` | 启动采集 | `{url}` | `{task_id, status}` |
| GET | `/api/collect/tasks` | 获取历史采集任务 | - | `[{task_id, status, title, ...}]` |
| GET | `/api/collect/<task_id>/status` | 查询采集状态 | - | `{status, progress, message, result}` |
| GET | `/api/collect/<task_id>/result` | 获取采集结果 | - | `{summary, product_data, images_mapping}` |
| POST | `/api/collect/<task_id>/open_folder` | 打开采集文件夹 | - | `{status, folder}` |
| GET | `/api/collect/<task_id>/product_status` | 查询是否已保存为正式产品 | - | `{saved, skc, skus, category}` |
| POST | `/api/collect/<task_id>/save_product` | 保存为正式产品 | - | `{success, skc, skus, category}` |

## 七、用户场景

### 场景1：批量生成产品图

1. 用户从Amazon采集到4张产品图
2. 使用AI工具生成4个俄文prompt
3. 打开sERP，新建任务"钱包产品图"
4. 拖拽4张图片到上传区，自动按文件名排序
5. 粘贴JSON配置，点击"拆分提示词"
6. 逐个点击"执行生图"或批量生成
7. 查看结果，不满意则重试
8. 下载满意的图片，用于Ozon上架

### 场景2：修改prompt重试

1. 打开已有任务
2. 修改某个卡片的prompt文本
3. 点击"重试"重新生成
4. 旧图片自动移到draft文件夹
5. 查看新生成的图片

### 场景3：多任务管理

1. 同时处理多个产品线
2. 创建任务"钱包产品图"、"支架产品图"
3. 随时切换任务继续处理
4. 所有数据自动保存

### 场景4：采集产品并保存为正式产品

1. 用户找到 Amazon 产品链接
2. 粘贴到采集输入框，点击"开始采集"
3. 等待采集完成（自动抓取标题、价格、描述、图片）
4. 查看采集数据，确认无误
5. 点击"保存为正式产品"
6. 系统自动分配 SKC/SKU 编码
7. 刷新页面后，卡片显示"✅ 已保存 (WALLET-0001)"
