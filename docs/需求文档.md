> AI 图像批量处理工具 + 电商产品数据采集工具
> 版本：v1.1
> 更新时间：2026-04-25

---

## 一、产品概述

sERP 是一款面向电商运营的 AI 图像批量处理工具，支持批量导入图片和 Prompt，调用 AI 生图 API 批量生成产品图。

### 目标用户
- 跨境电商卖家
- 需要批量处理产品图的运营人员

### 核心价值
- 批量上传 + 批量 Prompt 匹配
- 任务管理 + 数据持久化
- 提升图片处理效率

### 2.2 功能模块

#### 模块1：任务管理（左侧边栏）

| 功能 | 描述 |
|------|------|
| 任务列表 | 显示所有任务，按创建时间排序 |
| 新建任务 | 点击"+新建"按钮创建新任务 |
| 任务命名 | 双击任务名称可编辑重命名 |
| 切换任务 | 点击任务切换到对应任务界面 |
| 删除任务 | 点击"x"删除任务（需确认） |

**数据持久化要求：**
- 每个任务独立存储为一个 JSON 文件
- 文件路径：`tasks/{task_id}.json`
- 任务数据结构：
```json
{
  "id": "abc123",
  "name": "任务名称",
  "created_at": "2026-04-24T10:00:00",
  "json_input": "原始JSON文本",
  "cards": [
    {
      "imageName": "product1.jpg",
      "prompt": "提示词内容",
      "uploadUrl": "/uploads/xxx.jpg",
      "uploadPath": "/uploads/xxx.jpg",
      "resultUrl": "/outputs/abc123/card_0_xxx.png"
    }
  ]
}
```

---

#### 模块2：工具栏（顶部）

| 组件 | 功能 |
|------|------|
| JSON文本框1 | 填写JSON格式的图片配置，高度约96px |
| 拖拽上传区 | 支持拖拽或点击上传多张图片 |
| 拆分提示词按钮 | 解析JSON，将prompt填充到对应的文本框2 |

**JSON格式示例：**
```json
[
  {
    "image_name": "product1.jpg",
    "prompt": "产品图描述..."
  },
  {
    "image_name": "product2.jpg", 
    "prompt": "产品图描述..."
  }
]
```

---

#### 模块3：卡片区域（主内容区）

每个卡片包含：

| 组件 | 功能 |
|------|------|
| 图片框1 | 显示上传的原始图片，支持拖拽/点击上传 |
| 执行生图按钮 | 调用AI API生成图片 |
| 文本框2 | 显示/编辑prompt，约3-4行高度 |
| 图片框2 | 显示生成的结果图片 |
| 下载按钮 | 下载结果图片 |
| 重试按钮 | 重新生成图片 |

**卡片操作：**
1. 单独上传图片到某个图片框1
2. 编辑文本框2的prompt内容
3. 点击"执行生图"调用API
4. 查看结果图片，可下载或重试

---

#### 模块4：图片预览增强

| 功能 | 描述 |
|------|------|
| 悬停大图 | 鼠标移动到图片上显示320x320大图预览 |
| 右键新标签页 | 支持Chrome右键"在新标签页打开图片" |
| 点击跳转 | 点击图片在新标签页打开原图 |

---

#### 模块5：文件管理

| 目录 | 用途 |
|------|------|
| `uploads/` | 上传的原始图片 |
| `outputs/{task_id}/` | 每个任务的生成结果 |
| `outputs/{task_id}/draft/` | 重试后被丢弃的图片 |

**重试逻辑：**
1. 点击"重试"按钮
2. 旧结果图片移动到 `draft/` 子文件夹
3. 生成新图片
4. 更新结果图片URL

---

### 2.3 实时持久化

**触发保存的操作：**
- ✅ 创建任务
- ✅ 重命名任务
- ✅ 上传图片
- ✅ 编辑prompt文本框
- ✅ 生成图片成功
- ✅ 切换任务前（自动保存当前任务）

**防抖机制：**
- 文本编辑使用800ms防抖，避免频繁写入

## 三、API 接口需求

| 方法 | 路径 | 功能 | 请求体 | 响应 |
|------|------|------|--------|------|
| GET | `/api/tasks` | 获取任务列表 | - | `[{id, name, created_at}]` |
| POST | `/api/tasks` | 创建任务 | `{name?}` | `{id, name, created_at, cards}` |
| GET | `/api/tasks/<id>` | 获取任务详情 | - | 完整任务对象 |
| PATCH | `/api/tasks/<id>` | 更新任务 | `{name?, cards?, json_input?}` | 更新后的任务对象 |
| DELETE | `/api/tasks/<id>` | 删除任务 | - | `{ok: true}` |
| POST | `/api/upload` | 上传图片 | FormData: file | `{path, filename}` |
| POST | `/api/generate` | 生成图片 | `{prompt, image_path?, task_id, card_index}` | `{url, local_path}` |

## 五、用户场景

### 场景1：批量生成产品图

1. 用户从Amazon采集到4张产品图
2. 使用AI工具生成4个俄文prompt
3. 打开sERP，新建任务"钱包产品图"
4. 拖拽4张图片到上传区，自动按文件名排序
5. 粘贴JSON配置，点击"拆分提示词"
6. 逐个点击"执行生图"或批量生成
7. 查看结果，不满意则重试
8. 下载满意的图片，用于Ozon上架

### 场景2：修改prompt重试

1. 打开已有任务
2. 修改某个卡片的prompt文本
3. 点击"重试"重新生成
4. 旧图片自动移到draft文件夹
5. 查看新生成的图片

### 场景3：多任务管理

1. 同时处理多个产品线
2. 创建任务"钱包产品图"、"支架产品图"
3. 随时切换任务继续处理
4. 所有数据自动保存
